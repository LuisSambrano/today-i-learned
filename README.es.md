## Descripción General

Este repositorio sirve como una base de conocimiento personal donde documento conceptos técnicos, herramientas e insights encontrados durante el desarrollo de software. Cada entrada representa un momento discreto de aprendizaje, organizado por tema y fecha.

El formato sigue el patrón "Today I Learned" (TIL) popularizado en la comunidad de desarrolladores, donde notas cortas y enfocadas capturan conocimiento específico en lugar de tutoriales completos. Este enfoque crea una biblioteca de referencia buscable que crece orgánicamente con el tiempo.

Las entradas cubren una amplia gama de temas incluyendo lenguajes de programación, frameworks, diseño de sistemas, herramientas de desarrollo y mejores prácticas. El repositorio funciona tanto como referencia personal como recurso público para otros aprendiendo tecnologías similares.

## Formato

Cada entrada sigue una estructura consistente:

```markdown
# [Título del Tema]

**Fecha**: YYYY-MM-DD  
**Categoría**: [Nombre de Categoría]

## Lo que Aprendí

[Breve explicación del concepto o técnica]

## Ejemplo

\`\`\`[lenguaje]
// Código ejemplo demostrando el concepto
\`\`\`

## Por qué Importa

[Aplicación práctica o beneficio]

## Referencias

- [Enlace a documentación]
- [Enlace a artículo o tutorial]
```

## Categorías

Las categorías actuales incluyen:

- **JavaScript/TypeScript**: Características del lenguaje, patrones y mejores prácticas
- **React/Next.js**: Patrones de componentes, hooks, optimización de rendimiento
- **Bases de Datos**: Consultas SQL, diseño de esquemas, técnicas de optimización
- **Herramientas**: Git, VS Code, utilidades de terminal, flujos de desarrollo
- **Arquitectura**: Diseño de sistemas, patrones, consideraciones de escalabilidad
- **DevOps**: Deployment, CI/CD, infraestructura como código
- **Web Performance**: Técnicas de optimización, Core Web Vitals
- **Accesibilidad**: Pautas WCAG, patrones ARIA, diseño inclusivo

## Contribuir

Aunque esto es principalmente una base de conocimiento personal, las contribuciones son bienvenidas:

1. **Correcciones**: Si encuentras un error, abre un issue o pull request
2. **Adiciones**: Siéntete libre de sugerir recursos relacionados o enfoques alternativos
3. **Discusiones**: Usa GitHub Discussions para compartir tus propios aprendizajes

## Uso

### Navegación

Navega por los directorios para encontrar temas de interés. Cada archivo markdown es una entrada independiente.

### Búsqueda

Usa la funcionalidad de búsqueda de GitHub para encontrar temas específicos:

```
repo:LuisSambrano/til [término de búsqueda]
```

### Configuración Local

Clona el repositorio para navegar offline:

```bash
git clone https://github.com/LuisSambrano/til.git
cd til
```
