## Visão Geral

Este repositório serve como uma base de conhecimento pessoal onde documento conceitos técnicos, ferramentas e insights encontrados durante o desenvolvimento de software. Cada entrada representa um momento discreto de aprendizado, organizado por tema e data.

O formato segue o padrão "Today I Learned" (TIL) popularizado na comunidade de desenvolvedores, onde notas curtas e focadas capturam conhecimento específico em vez de tutoriais completos. Esta abordagem cria uma biblioteca de referência pesquisável que cresce organicamente ao longo do tempo.

As entradas cobrem uma ampla gama de tópicos incluindo linguagens de programação, frameworks, design de sistemas, ferramentas de desenvolvimento e melhores práticas. O repositório funciona tanto como referência pessoal quanto recurso público para outros aprendendo tecnologias similares.

## Formato

Cada entrada segue uma estrutura consistente:

```markdown
# [Título do Tópico]

**Data**: YYYY-MM-DD  
**Categoria**: [Nome da Categoria]

## O que Aprendi

[Breve explicação do conceito ou técnica]

## Exemplo

\`\`\`[linguagem]
// Código exemplo demonstrando o conceito
\`\`\`

## Por que Importa

[Aplicação prática ou benefício]

## Referências

- [Link para documentação]
- [Link para artigo ou tutorial]
```

## Categorias

As categorias atuais incluem:

- **JavaScript/TypeScript**: Características da linguagem, padrões e melhores práticas
- **React/Next.js**: Padrões de componentes, hooks, otimização de performance
- **Bancos de Dados**: Consultas SQL, design de schema, técnicas de otimização
- **Ferramentas**: Git, VS Code, utilitários de terminal, fluxos de desenvolvimento
- **Arquitetura**: Design de sistemas, padrões, considerações de escalabilidade
- **DevOps**: Deploy, CI/CD, infraestrutura como código
- **Web Performance**: Técnicas de otimização, Core Web Vitals
- **Acessibilidade**: Diretrizes WCAG, padrões ARIA, design inclusivo

## Uso

### Navegação

Navegue pelos diretórios para encontrar tópicos de interesse. Cada arquivo markdown é uma entrada independente.

### Busca

Use a funcionalidade de busca do GitHub para encontrar tópicos específicos:

```
repo:LuisSambrano/til [termo de busca]
```

### Configuração Local

Clone o repositório para navegar offline:

```bash
git clone https://github.com/LuisSambrano/til.git
cd til
```
